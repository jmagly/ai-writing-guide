## Purpose

A boilerplate library provides standardized company descriptions in various lengths for press releases, media kits, website footers, email signatures, pitch decks, and other materials requiring consistent brand messaging.

## How to Use This Library

1. Select the appropriate length for your context
2. Copy the exact text (do not modify without brand team approval)
3. Update metrics quarterly to keep boilerplates current
4. Add attribution footer when required (e.g., press releases)
5. Use specialized variations for specific audiences (investor vs. customer)

## 6. Update Triggers & Process

### 6.1 When to Update Boilerplates

**Quarterly** (scheduled):
- Refresh metrics (customer count, revenue, funding)
- Update "since [year]" references
- Verify customer names (get permission for new names)

**Milestone-Triggered** (as they occur):
- New funding round announced
- Major product launch
- Significant customer milestone (e.g., 10,000 customers)
- Geographic expansion
- New notable customer wins
- Awards or recognition

**Annual** (major review):
- Revise positioning if market shifted
- Update mission/vision if evolved
- Refresh problem statement if market education changed

### 6.2 Approval Process

**Minor Updates** (metrics only):
- Update and notify team
- Approver: [Marketing Manager]

**Major Updates** (positioning, messaging):
- Draft update
- Review with brand team
- Approver: [CMO or Brand Director]
- Communicate changes to all teams using boilerplates

### 6.3 Distribution After Updates

When boilerplates are updated, notify:
- [ ] Sales team (update pitch decks, proposals)
- [ ] Marketing team (update website, press kit)
- [ ] Communications team (update press release templates)
- [ ] HR/Recruiting (update job postings, career site)
- [ ] Partnerships team (update partnership materials)

## 7. Usage Guidelines

### 7.1 When to Use Each Length

| Length | Context | Example Use Case |
|--------|---------|-----------------|
| **Ultra-Short** (25-35 words) | Space-constrained | Twitter bio, email signature, app store |
| **Short** (50-75 words) | Brief overview | Website footer, LinkedIn About, pitch deck closer |
| **Medium** (100-150 words) | Standard description | Press releases, media kit, About page summary |
| **Long** (200-250 words) | Comprehensive overview | Detailed About page, investor appendix, award applications |

### 7.2 Customization Rules

**Allowed**:
- ✓ Select appropriate length for context
- ✓ Choose audience-specific variation (investor vs. customer)
- ✓ Update metrics if more current data available (with approval)
- ✓ Translate to other languages (with native speaker review)

**Not Allowed**:
- ✗ Edit wording without brand team approval
- ✗ Mix components from different audience variations
- ✗ Add claims not in approved boilerplates
- ✗ Remove proof points to save space (use shorter version instead)

### 7.3 Quality Checklist

Before publishing boilerplate, verify:

- [ ] Correct length selected for context
- [ ] Metrics are current (within last quarter)
- [ ] Customer names have permission for public use
- [ ] Trademark symbol (®) included on first company name use if applicable
- [ ] Website URL is correct and includes https://
- [ ] No typos or grammatical errors
- [ ] Formatting matches brand standards
- [ ] Contact info is current (if included)
